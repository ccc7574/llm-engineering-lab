# P12: 做 Context Length 扩展实验

## 背景

上下文长度不是单纯把参数改大这么简单，它会影响训练稳定性、显存占用、样本组织和真实可用性。

## 任务目标

- 比较不同 block size 下的训练和采样行为
- 理解长上下文带来的收益和成本
- 学会判断“窗口更长”是否真的等于“能力更强”

## 业务与资源约束

- 只做 tiny 级别上下文扩展
- 必须同时看收益和成本

## 代码改造点

- `code/stage1_nanogpt_core/train.py`
- `code/stage1_nanogpt_core/sample.py`

## 交付标准

- 至少比较两档 context window
- 记录训练成本与输出差异
- 能解释长上下文何时值得、何时只是昂贵装饰

## 原理补充

长上下文真正有价值的前提是数据和任务确实需要它，否则往往只会增加成本和训练不稳定性。

## 实验步骤

1. 调整 `block_size`
2. 用相同任务做短跑训练
3. 对比长短窗口下的输出

## 结果解读

如果任务本身依赖远距离信息，长窗口才会明显有益；否则更多只是容量冗余。

## 线上风险

- 显存和时延开销被低估
- 训练样本长度分布与真实任务不匹配
