# P02: 证明模型在利用上下文

## 背景

很多人会说 transformer “能看上下文”，但如果不能通过实验看到上下文窗口变化如何改变输出，这句话对工程师没有操作价值。

## 任务目标

- 用对照提示验证模型确实在利用前文
- 观察 context window 改变后的输出差异
- 用可视化建立对 attention 的直觉

## 业务与资源约束

- 只做小样本、可解释实验
- 不要求数学推导完整

## 代码改造点

- `code/stage1_nanogpt_core/sample.py`
- `code/stage1_nanogpt_core/visualize_attention.py`

## 交付标准

- 至少产出 2 组 `compare-start` 对照
- 至少产出 1 张 attention heatmap
- 能解释局部上下文和远距离上下文的差别

## 原理补充

这一单不是证明 attention “像人类理解”，而是证明模型输出依赖输入窗口中的 token 排布和权重分配。

## 实验步骤

1. 用不同起始提示做对照采样
2. 改 `context-window` 观察差异
3. 导出 attention heatmap

## 结果解读

如果两个提示只改了关键上下文就导致生成显著变化，说明模型已经在利用条件信息，而不是纯随机续写。

## 线上风险

- 把相关性误当成真实理解
- 只看单个可视化样例，结论过强
