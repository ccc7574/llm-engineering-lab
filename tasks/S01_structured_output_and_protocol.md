# S01: 让结构化输出和协议稳定下来

## 背景

很多业务问题不是“模型会不会回答”，而是“它能不能持续按我们要求的协议返回”。结构化输出是后训练最常见也最容易踩坑的场景。

## 任务目标

- 设计稳定的输出模板
- 验证 prompt template 与 label mask 是否一致
- 比较自由回答与结构化回答的稳定性

## 业务与资源约束

- 必须可复现
- 输出格式要便于下游系统消费

## 代码改造点

- `code/stage2_sft/prompt_template.py`
- `eval/sft_eval.py`

## 交付标准

- 至少有一类结构化任务
- 能统计格式遵守率
- 能说明协议稳定性为何重要

## 原理补充

所谓结构化输出，本质上是把模型行为约束到某种固定协议里，让下游系统能可靠消费。

## 实验步骤

1. 增加结构化样本
2. 比较自由输出和结构化输出
3. 记录格式错误类型

## 结果解读

重点看稳定性，而不是只看少数漂亮样例。

## 线上风险

- 格式看似正确但字段语义错位
- 模板稍变就失稳
