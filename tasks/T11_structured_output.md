# T11: 让助手稳定输出结构化结果

## 背景

产品已经不满足于“看起来像回答”的文本。很多真实系统需要把模型输出接到工单流转、自动化脚本或审核面板上，这意味着模型必须稳定地产出 JSON、键值对或 checklist。

## 任务目标

- 为常见任务定义结构化输出模板
- 观察模型在自由文本和结构化格式上的差异
- 明确“格式稳定”是一类独立工程能力

## 业务约束

- 输出要便于机器消费
- 模板要尽量少但稳定
- 不能把结构化能力完全依赖在 prompt 运气上

## 你需要改的代码

- `code/stage2_sft/prompt_template.py`
- `code/stage2_sft/dataset.py`
- `eval/sft_eval.py`

## 交付标准

- 至少支持一个 JSON 输出任务
- 至少支持一个 checklist 输出任务
- 记录输出格式错误率

## 原理补充

工程上常见误区是把“模型会回答”误认为“模型能稳定接入系统”。结构化输出要解决的其实是协议问题，而不是单纯语言流畅度问题。
